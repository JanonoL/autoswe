# /sdlc-release —— 发布安全清单

把部署升级为完整发布工程。CI 全绿后由人确认影响面与回滚再上线。

## 生成清单（写入 `.planning/` 当前 phase 的 RELEASE.md）
- [ ] 渐进发布：灰度/金丝雀策略与放量步骤。
- [ ] 回滚就绪：回滚脚本/补偿可执行；Schema 变更可回滚或有补偿。
- [ ] 特性开关：高风险功能一键关闭，无需回滚部署。
- [ ] 上线监控：关键业务指标、错误日志、慢请求阈值、告警项。
- [ ] 影响面确认：引用 /sdlc-impact 与 TRACE.md 发布影响列。
- [ ] 双门绿证：引用 /sdlc-gates 两门皆绿结果。
- [ ] 人工裁决点：谁在何条件下批准上线。

## 准出
未勾满且无人工裁决签字 → 不得上线。
