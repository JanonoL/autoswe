# 规则：可追溯链

**目标**：需求—设计—代码—测试—发布不断链，支撑变更增量传播。

## 追溯矩阵字段
| 字段 | 含义 |
|------|------|
| 需求 ID | 唯一标识 |
| 验收用例 | Gherkin / 用例文件路径 |
| API 契约 | OpenAPI 路径 |
| 涉及模块/接口 | 实现位置 |
| 核心测试 | 单测/契约/API/E2E 清单 |
| 数据/Schema 变更 | 迁移脚本 |
| 发布影响 | 影响面 |
| 回滚方案 | 回退/补偿方式 |

## 落盘
追溯矩阵随 gsd 工件落 `.planning/`（如 phase 目录或 SDLC-TRACE.md）。可用 `/sdlc-trace` 生成/更新。

## 变更增量传播
需求变更时以本矩阵为索引，只重生成/重跑受影响部分，不全量重跑。
