# 规则：改动前影响分析门（演进场景）

**目标**：改存量必先定位影响集，防止破坏既有调用方。

## 强制步骤（改既有接口/方法/字段前）
1. 全量定位调用方：**优先用 codegraph 调用图**（MCP `codegraph_explore`，或 CLI `codegraph callers/impact/affected`）精确定位——跨模块/接口/Feign 间接调用都能查到；无 codegraph 时回退 grep 全量搜索。
2. 逐调用方评估：它依赖什么行为？本次改动是否破坏？
3. 选最安全策略：优先高层(Service)改动而非底层共享方法；优先新增而非改既有签名；必须改签名则保证向后兼容。
4. 标记回归保护集：受影响模块必须重跑的既有测试清单。

## 准出
任一调用方可能被破坏且无安全策略 → 停止改动，升级人工。可用 `/sdlc-impact` 命令生成影响报告。
