# 规则：发布安全

**目标**：部署 ≠ 生成脚本；CI 全绿后由人确认影响面与回滚再上线。

## 发布清单（Ship 阶段必勾）
- [ ] 渐进发布：灰度/金丝雀，先小范围验证（比例与放量步骤）。
- [ ] 回滚就绪：回滚脚本/补偿方案可执行；Schema 变更可回滚或有补偿（双写/兼容窗口）。
- [ ] 特性开关：高风险功能可一键关闭，无需回滚部署。
- [ ] 上线监控：关键业务指标、错误日志、慢请求阈值、告警项。
- [ ] 影响面确认：引用影响分析与追溯矩阵的发布影响列。
- [ ] 双门绿证：引用双验证门两门皆绿结果。
- [ ] 人工裁决点：谁在何条件下批准上线。

## 准出
未勾满且无人工裁决签字 → 不得上线。可用 `/sdlc-release` 生成清单。
