# 规则：双验证门（功能门 + 非功能门）

**目标**：功能对不对 + 软件工程的另一半（安全/性能/可维护性）并行把关。两门皆绿才放行。

## 功能门
单元/集成/契约/API/E2E 按层运行项目实际命令，收集退出码。

## 非功能门(NFR)
| 维度 | 门禁内容 |
|------|----------|
| 安全 | SAST 无高危；依赖无已知高危漏洞；无密钥/敏感数据提交；越权/鉴权负向测试通过 |
| 性能 | 关键接口压测达基线；无明显 N+1/全表扫描；复杂度不超阈值 |
| 可维护性 | 重复率/圈复杂度达标；分层依赖与模块边界未被破坏 |
| 供应链 | 新增依赖说明用途/许可证/维护状态/替代方案 |

## 铁律（verifier 强制）
- 裁判是确定性工具（测试/扫描/构建），**AI 不自评**。
- **禁止**为换绿而关闭鉴权、放宽校验、跳过或弱化任何门。
- 任一门红 → 自愈（见 self-healing-cap），不得直接放行。

## gsd config 对应开关
`workflow.verifier:true`、`workflow.security_enforcement:true`、`workflow.security_block_on:"high"`、`workflow.code_review_command` 指向项目 lint/扫描命令。
