# 规则：需求质量矩阵（7 类必覆盖）

**目标**：把"用例写全"变成可校验的方法，堵住隐含需求/边界/NFR 遗漏。

## 最低覆盖矩阵（缺一不可）
| 类型 | 必须覆盖 |
|------|----------|
| 正常路径 | ≥1 条主流程 |
| 异常路径 | 参数错误、状态错误、权限错误 |
| 边界条件 | 时间/数量/金额/分页边界 |
| 权限 | 无权限、越权、租户/数据隔离 |
| 一致性 | 事务、幂等、重复提交、并发冲突 |
| 审计 | 操作日志、业务流水、追踪 ID |
| 兼容性 | 老数据、老接口、字段默认值 |

## 对抗式补全（Discuss 阶段）
进规划前 AI 主动追问并由人裁决：异常路径？边界在哪？并发/幂等？权限？NFR？老数据兼容？

## 校验点（plan-checker）
需求/计划缺失上述任一类覆盖 → 标注缺口，要求补全后再进入执行。
