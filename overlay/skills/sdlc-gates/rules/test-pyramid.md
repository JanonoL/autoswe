# 规则：测试金字塔

**目标**：健康的测试形状，不堆 E2E（避免倒金字塔/冰淇淋甜筒反模式）。

## 分层占比（建议）
- 单元 ~70%（毫秒级）：逻辑/分支/边界。
- 集成 + 契约 ~20%（秒级）：组件协作、消费者-提供者兼容。
- API/BDD ~7%（秒~分）：接口级验收。
- E2E ~3%��分钟级）：仅关键用户旅程。

## 校验点（plan-checker / verifier）
- 计划为每个用户故事标注测试归属层级。
- 能在低层覆盖的逻辑，不得新增 E2E 任务覆盖。
- 测试套件总时长纳入度量，超阈值要治理（拆分/并行/下沉低层）。
- 发现"高层测试覆盖了本该低层测的逻辑" → 判定不通过，要求下沉。
